Run a shell command as a background task.

Use this instead of `bash` when:
- The command needs to keep running while you do other work
  (dev servers, watch mode, long-running APIs)
- You need to send interactive input (Ctrl+C, arrow keys, etc.)
- You need to run multiple commands concurrently

Use `bash` (the built-in tool) for one-shot commands where you only
need to see the output once.

The command runs in a PTY, allowing proper TTY behavior like colored
output and signal handling.

Usage:
- `command`: Required — the executable to run (e.g., "npm", "python", "bash")
- `args`: Arguments to pass to the command (e.g., ["run", "dev"])
- `workdir`: Working directory (defaults to project root)
- `env`: Additional environment variables
- `title`: Human-readable title for the session
- `description`: Required — clear, concise 5-10 word description of what this task does
- `timeoutSeconds`: Auto-kill after N seconds
  - Do NOT set for persistent tasks (dev servers, watch mode, REPLs)
  - DO set for tasks that should finish on their own (builds, tests, migrations)

Returns session info including:
- `id`: Unique identifier (pty_XXXXXXXX) for use with other pty_* tools
- `pid`, `status`, `command`, `workdir`

After spawning, use:
- `pty_write` to send input to the task
- `pty_read` to read output from the task
- `pty_list` to see all active tasks
- `pty_status` to check if a task is still running
- `pty_await` to wait for a task to finish
- `pty_kill` to stop a task

Examples:
- Start a dev server: command="npm", args=["run", "dev"], title="Dev Server"
- Start a timed dev server: command="npm", args=["run", "dev"], title="Dev Server", timeoutSeconds=600
- Start a Python REPL: command="python3", title="Python REPL"
- Run tests in watch mode: command="npm", args=["test", "--", "--watch"]
- Run build with timeout: command="npm", args=["run", "build"], title="Build", timeoutSeconds=900
- Run end-to-end tests with timeout: command="npm", args=["run", "test:e2e"], title="E2E Tests", timeoutSeconds=1800
