Wait for a background task to finish and return its exit status.

Blocks until the task exits, is killed, or the timeout is reached.
Use this when you need to wait for a task like a build or test suite
to complete before reading its output.

Parameters:
- `id`: The PTY session ID returned by pty_spawn
- `timeoutSeconds`: Maximum wait time in seconds (default: 60)

Behavior:
- Blocks until the session exits, is killed, or timeout is reached
- When timeout is reached, returns state: "running" — the task is still alive
- Once the session exits, the exit code and output line count are available

Returns:
- `session_id`: The PTY session ID
- `state`: exited | killed | running (timeout reached)
- `exit_code`: Exit code (if exited)
- `signal`: Signal number (if killed)
- `line_count`: Number of output lines in the buffer

After awaiting, use `pty_read` to inspect the full output.

Examples:
- Wait for a build to finish: id="pty_a1b2c3d4"
- Wait with custom timeout: id="pty_a1b2c3d4", timeoutSeconds=120

Use `pty_status` if you only need to check the current state without blocking.
